## Context

The backoffice currently has no screen to monitor `AiBatchJob` execution. Admins need to:
- Identify which AI job is running (workflow type, step, external OpenAI/Claude batch ID)
- Track progress (requestCount / succeededCount / failedCount)
- Filter by status and workflow type

Entities `AiBatchJob` and `AiBatchJobItem` already exist — no new backend model required.
The work is purely a new EasyAdmin CRUD controller + Twig template + dashboard menu entry.

### Existing entities (read-only, already in DB)

**`App\Entity\OpenAI\AiBatchJob`**
- `id`, `createdAt`, `updatedAt`
- `workflowType` — values: `stamp`, `philately`, `collection`
- `step` — numeric, matches prompt step 501–511
- `status` — `pending` | `processing` | `ended` | `failed` | `in_progress` | `canceling`
- `processingStatus`
- `requestCount`, `succeededCount`, `failedCount`
- `externalId` — OpenAI/Claude batch ID
- `cacheWriteTokens`, `cacheReadTokens`
- OneToMany → `aiBatchJobItems`

**`App\Entity\OpenAI\AiBatchJobItem`**
- `id`, `step`, `status` (`pending` | `succeeded` | `errored` | `skipped`)
- `customId` (format `pt_{productId}_step_{step}`)
- `outMessages`, `resultData`, `errorMessage`
- ManyToOne → `AiBatchJob`

## Implementation steps

### 1. `AiBatchJobCrudController.php`

```php
namespace App\Controller\Admin\OpenAI;

use App\Entity\OpenAI\AiBatchJob;
use EasyCorp\Bundle\EasyAdminBundle\Controller\AbstractCrudController;
use EasyCorp\Bundle\EasyAdminBundle\Config\Crud;
use EasyCorp\Bundle\EasyAdminBundle\Config\Actions;
use EasyCorp\Bundle\EasyAdminBundle\Config\Filters;
use EasyCorp\Bundle\EasyAdminBundle\Field\...;
```

**`configureCrud()`**:
- Label: `"Batch IA"`
- Default sort: `createdAt DESC`
- Override template: `'crud/index' => 'admin/openai-batch/crud/index.html.twig'`

**`configureActions()`**:
- Disable `new`, `edit`, `delete` (read-only monitoring screen)
- Keep `detail` if useful

**`configureFilters()`**:
- `status` (ChoiceFilter — values: pending, processing, ended, failed, in_progress, canceling)
- `workflowType` (ChoiceFilter — values: stamp, philately, collection)
- `createdAt` (DateTimeFilter)

**`configureFields()`** for INDEX:
- `IdField::new('id')`
- `DateTimeField::new('createdAt', 'Créé le')`
- `TextField::new('workflowType', 'Workflow')`
- `IntegerField::new('step', 'Étape')`
- `TextField::new('status', 'Statut')` (custom badge via template)
- `TextField::new('processingStatus', 'Statut traitement')`
- `IntegerField::new('requestCount', 'Requêtes')`
- `IntegerField::new('succeededCount', 'Succès')`
- `IntegerField::new('failedCount', 'Échecs')`
- `TextField::new('externalId', 'ID externe')`

### 2. `templates/admin/openai-batch/crud/index.html.twig`

Extends `@EasyAdmin/crud/index.html.twig`.

Custom cells:
- **Status** column: coloured badge (`<span class="badge badge-...">`) per status value
  - `pending` → secondary, `processing`/`in_progress` → primary, `ended` → success, `failed`/`canceling` → danger
- **Progress** column (extra computed column): `{{ (row.succeededCount + row.failedCount) ~ '/' ~ row.requestCount }}`
  with a Bootstrap `progress-bar` showing `((succeededCount + failedCount) / requestCount * 100)%`

### 3. `DashboardController.php`

In the menu configuration, add inside the OpenAI section (or create one if missing):
```php
MenuItem::linkToCrud('Batch IA', 'fa fa-robot', AiBatchJob::class),
```

## Notes & constraints

- **Prompt steps 501–511 must NOT be renumbered** (active production batches depend on these IDs)
- `AiBatchJob.step` is a numeric field matching these prompt IDs — display as-is
- If `requestCount = 0`, guard against division by zero in the progress bar Twig expression
- Access control: restrict to `ROLE_ADMIN` (same as other OpenAI admin screens)
